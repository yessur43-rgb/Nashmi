# ZAD | زاد - رفيقك الذكي في السفر

**زاد** هو تطبيق ويب متقدم ومصمم خصيصًا للمسافرين، لا سيما المسلمين، ليكون رفيقهم الذكي في كل رحلة. مدعومًا بواجهة برمجة تطبيقات Google Gemini، يقدم التطبيق مجموعة قوية من الأدوات التي تجعل السفر أسهل وأكثر متعة وثقة. التطبيق مصمم للعمل دون اتصال بالإنترنت أولاً (Offline-First) مع الحفاظ على خصوصية بيانات المستخدم بالكامل عن طريق تخزينها محليًا على الجهاز.

## 🧭 فلسفة المشروع

-   **الذكاء الاصطناعي أولاً (AI-First):** تم تصميم كل أداة للاستفادة من قدرات نماذج Gemini المتقدمة لتقديم إجابات وتحليلات ذكية ودقيقة.
-   **الخصوصية والعمل دون اتصال (Privacy & Offline-First):** يتم تخزين جميع بيانات المستخدم (الرحلات، اليوميات، الصور، المفضلة) بشكل آمن في **IndexedDB** على متصفح المستخدم. هذا يضمن خصوصية كاملة ووصولاً للبيانات حتى بدون اتصال بالإنترنت.
-   **تجربة مستخدم سلسة (Seamless UX):** واجهة بسيطة وواضحة باللغة العربية، مصممة لتكون سهلة الاستخدام أثناء التنقل.

## ✨ الميزات الأساسية

يحتوي "زاد" على مجموعة متكاملة من الأدوات الذكية لتلبية احتياجاتك أثناء السفر:

-   **تحليل المنتج (Product Analyzer):**
    -   تحقق مما إذا كانت المنتجات **حلالاً** أو **حراماً** أو **مشبوهة**.
    -   يدعم التحليل عبر **كاميرا الهاتف**، **مسح الباركود**، أو **البحث باسم المنتج**.
    -   يقدم تقريراً مفصلاً يتضمن المكونات، سبب الحكم، معلومات صحية، وحقائق غذائية.

-   **تحليل القائمة (Menu Analyzer):**
    -   التقط صورة لقائمة طعام في أي مطعم، وسيقوم التطبيق بتحليل الأطباق وتحديد حالتها الشرعية.

-   **مساحتي (My Space):**
    -   دفتر يوميات سفر شخصي وذكي.
    -   أضف الصور، الفيديوهات، الملاحظات الصوتية، والمصاريف اليومية.
    -   يقوم الذكاء الاصطناعي بتحليل الوسائط تلقائيًا وكتابة أوصاف ذكية لها.
    -   يمكنك تحويل يوميات رحلتك إلى **قصة متكاملة وجذابة** بتنسيق HTML بضغطة زر.

-   **البحث والاكتشاف:**
    -   **ابحث عن أماكن (Find Places):** اكتشف المطاعم الحلال، المساجد، وغيرها بالقرب منك.
    -   **أوجدها لي (Find It):** ابحث عن منتجات أو متاجر معينة بالبحث النصي أو بالصور.
    -   **على طريقي (On My Way):** ابحث عن محطات توقف مناسبة على طول طريق رحلتك.
    -   **الأنشطة (Activities Finder):** ابحث عن أنشطة وفعاليات مناسبة للعائلة.
    -   **أين قلب المدينة؟ (City Center Finder):** استكشف مراكز المدن الحيوية القريبة.
    -   **باحث الأمطار (Rain Finder):** اعثر على الأماكن التي تمطر فيها حاليًا بالقرب منك.

-   **أدوات مساعدة:**
    -   **أين أوقفت سيارتي؟ (Park My Car):** احفظ موقع سيارتك بصورة ووصف ذكي.
    -   **أين أسكن؟ (My Accommodation):** احفظ مواقع أماكن سكنك للعودة إليها بسهولة.
    -   **دليل المكونات (Ingredient Guide):** احصل على معلومات مفصلة حول المكونات الغذائية وأرقام E-numbers.
    -   **المفضلة (Favorites):** احفظ أماكنك وأنشطتك المفضلة.
    -   **اسألني أي شيء (Ask Me Anything):** مساعد سفر محادثة للإجابة على جميع أسئلتك.

## 🏗️ المعمارية التقنية

يعتمد التطبيق على معمارية من جانب العميل (Client-Side) بالكامل، حيث تحدث جميع عمليات العرض والتخزين والتفاعل في متصفح المستخدم.

```
+--------------------------------+
|      المستخدم (المتصفح)         |
+--------------------------------+
|      React UI Components       | (components/)
| (ProductAnalyzer, MySpace, etc.) |
+--------------------------------+
|         App.tsx (State         | (إدارة الحالة والتنقل)
|      Management & Routing)     |
+--------------------------------+
|      Services (الخدمات)         |
| +----------------------------+ |
| | geminiService.ts           | | (تجريد استدعاءات Gemini API)
| +----------------------------+ |
| | dbService.ts               | | (تجريد عمليات IndexedDB)
| +----------------------------+ |
+--------------------------------+
|      Browser APIs (واجهات     | (Geolocation, Camera, etc.)
|     برمجة تطبيقات المتصفح)     |
+--------------------------------+
|      IndexedDB (Local         | (تخزين دائم للبيانات)
|         Database)              |
+--------------------------------+
```

## 📁 هيكل المشروع

تم تنظيم الشفرة المصدرية في وحدات واضحة لسهولة الصيانة والتطوير.

```
/
├── public/                     # الأصول الثابتة
├── index.html                  # نقطة الدخول الرئيسية لـ HTML
├── index.tsx                   # نقطة دخول React الرئيسية
├── App.tsx                     # المكون الجذري للتطبيق، يدير الحالة والتنقل
├── types.ts                    # تعريفات TypeScript للأنواع المشتركة
├── declarations.d.ts           # تعريفات للواجهات غير القياسية (مثل BarcodeDetector)
├── components/
│   ├── common/                 # مكونات واجهة مستخدم مشتركة (Header, Button, etc.)
│   ├── ProductAnalyzer.tsx     # مكون أداة تحليل المنتج
│   ├── MySpace.tsx             # مكون أداة "مساحتي" (اليوميات والرحلات)
│   └── ... (مكون لكل أداة)
├── services/
│   ├── geminiService.ts        # وحدة مركزية لجميع استدعاءات Google Gemini API
│   └── dbService.ts            # طبقة تجريد (Abstraction Layer) للتعامل مع IndexedDB
└── utils/
    └── helpers.ts              # دوال مساعدة (مثل تحويل الصور إلى base64)
```

## ⚙️ آلية عمل الميزات الرئيسية

### 1. تحليل المنتج والوسائط (Gemini API)

-   يتم التعامل مع جميع التفاعلات مع Gemini API عبر `services/geminiService.ts`.
-   **المرونة:** تم تطبيق دالة `generateContentWithRetry` التي تقوم بإعادة المحاولة تلقائيًا عند حدوث أخطاء من الخادم (مثل `5xx` أو `429`) باستخدام خوارزمية **Exponential Backoff** لضمان استقرار التطبيق.
-   **الاستجابات المنظمة:** يتم فرض استجابات JSON منظمة من Gemini API باستخدام `responseSchema` في إعدادات الطلب، مما يضمن أن البيانات الواردة متوافقة دائمًا مع الأنواع المحددة في `types.ts`.
-   **النماذج المستخدمة:**
    -   `gemini-2.5-pro`: للمهام المعقدة التي تتطلب فهماً عميقاً وتحليلاً للصور والنصوص (مثل تحليل المنتجات، اليوميات، والقوائم).
    -   `gemini-2.5-flash`: للمهام السريعة مثل تفسير أوامر المستخدم، تصنيف النفقات، والإجابة على الأسئلة العامة.
    -   `gemini-2.5-flash-image`: لمهام تحرير الصور وتحسينها.

### 2. تخزين البيانات المحلي (IndexedDB)

-   يتم تجريد جميع عمليات قاعدة البيانات في `services/dbService.ts` لتسهيل التعامل مع IndexedDB.
-   **مخازن الكائنات (Object Stores):**
    -   `trips`: لتخزين جميع تفاصيل الرحلات واليوميات.
    -   `accommodations`, `parking`: لتخزين مواقع السكن ومواقف السيارات.
    -   `favorite_*`: مخازن متعددة لتخزين أنواع مختلفة من المفضلات.
    -   `settings`: لتخزين إعدادات المستخدم مثل مفتاح API ووضع المظهر.
-   هذا النهج يضمن أن التطبيق يعمل بكفاءة حتى بدون اتصال بالإنترنت، ويحافظ على خصوصية بيانات المستخدمين.

### 3. دفتر اليوميات الذكي (MySpace Workflow)

تعتبر ميزة "مساحتي" من أكثر الميزات تعقيدًا وتوضح قوة التطبيق:
1.  **الالتقاط:** يلتقط المستخدم صورة أو فيديو.
2.  **المعالجة الأولية:** يتم ضغط الصورة وإنشاء صورة مصغرة (thumbnail) لها في المتصفح عبر `utils/helpers.ts`.
3.  **التحليل الذكي:** يتم إرسال الصورة (كـ base64) مع الموقع الجغرافي إلى `geminiService.ts`.
4.  **التصنيف:** يقوم Gemini بتحديد ما إذا كانت الصورة لإيصال (فتصبح مصروفًا) أم صورة عادية.
5.  **استخراج البيانات:**
    -   **إذا كانت مصروفًا:** يستخرج Gemini المبلغ، العملة، والوصف ويحول المبلغ إلى الريال السعودي.
    -   **إذا كانت صورة عادية:** يكتب Gemini وصفًا قصصيًا وذكيًا للصورة بناءً على محتواها وموقعها الجغرافي.
6.  **التخزين:** يتم حفظ البيانات المستخرجة مع الصورة الأصلية في IndexedDB كجزء من يوميات الرحلة النشطة.
7.  **إنشاء القصة:** عند الطلب، يتم إرسال جميع يوميات الرحلة (نصوص، أوصاف صور) إلى Gemini لإنشاء قصة سفر متكاملة.
8.  **تصدير HTML:** يقوم Gemini بتنسيق القصة في ملف HTML أنيق وتفاعلي، مع تضمين الصور مباشرة في الملف (base64) ليصبح ملفًا مستقلاً يمكن مشاركته بسهولة.

## 🛠️ التقنيات المستخدمة

-   **الواجهة الأمامية (Frontend):**
    -   **React (v19):** مكتبة واجهة المستخدم الأساسية.
    -   **TypeScript:** لإضافة الأمان النوعي (Type Safety) للشفرة.
    -   **Tailwind CSS:** لإطار عمل تصميم سريع ومرن.
    -   **Lucide React:** لمجموعة أيقونات خفيفة وجميلة.

-   **الذكاء الاصطناعي (AI):**
    -   **`@google/genai` SDK:** المكتبة الرسمية للتفاعل مع Google Gemini API.

-   **قاعدة البيانات (Database):**
    -   **IndexedDB:** قاعدة بيانات مدمجة في المتصفح للتخزين المحلي.

-   **واجهات برمجة تطبيقات الويب (Web APIs):**
    -   Geolocation API: لتحديد الموقع.
    -   MediaDevices (Camera) API: لالتقاط الصور والفيديو.
    -   Barcode Detection API: لمسح الباركود.
    -   Speech Recognition API: للإدخال الصوتي.

## 🚀 إعداد وتشغيل المشروع

لتشغيل هذا المشروع محليًا للتطوير، اتبع الخطوات التالية.

### المتطلبات

-   [Node.js](https://nodejs.org/) (إصدار 18 أو أحدث)
-   [npm](https://www.npmjs.com/) أو [yarn](https://yarnpkg.com/)

### خطوات الإعداد

1.  **استنساخ المستودع (Clone Repository):**
    ```bash
    git clone <repository_url>
    cd <repository_directory>
    ```

2.  **تثبيت الاعتماديات (Install Dependencies):**
    ```bash
    npm install
    ```

3.  **إعداد متغيرات البيئة (Environment Variables):**
    سيحتاج التطبيق إلى مفتاح Google Gemini API. الطريقة المتبعة في هذا المشروع هي أن المستخدم يدخل المفتاح عبر الواجهة، ولكن في بيئة تطوير، يمكنك إعداده مسبقًا.

4.  **تشغيل خادم التطوير (Run Development Server):**
    ```bash
    npm run dev
    ```
    سيتم تشغيل التطبيق على `http://localhost:5173` (أو منفذ آخر إذا كان 5173 مشغولاً).
